# AAB-9 — Перенести PROJECT_CONTEXT.md в репозиторий

Не срочная, организационная задача — выполняется **после** AAB-8
(бывшая ANEX-008, аудит безопасности), не раньше.

## Цель

`PROJECT_CONTEXT.md` раньше жил только как вручную загруженный документ в
project knowledge Claude.ai Projects, отдельно от репозитория. Репозиторий
подключён к Project через GitHub-синк — переносим документ туда же, чтобы он
тоже шёл через этот синк, а не поддерживался вручную отдельно.

Архитектор уже подготовил обновлённое содержание документа под текущее
состояние проекта — готовый текст приложен отдельным файлом
`PROJECT_CONTEXT.md`, просто разместить его как есть, не переписывать заново.

## Порядок работ

1. Положить присланный архитектором `PROJECT_CONTEXT.md` в `docs/PROJECT_CONTEXT.md`.
2. Проверить, что в `CLAUDE.md` ссылка на этот документ по-прежнему верна по
   смыслу (документ теперь физически в репозитории, но по-прежнему адресован
   архитектору, а не сеньору — реализацию из него не тянуть напрямую, только
   через `AAB-N.md`).
3. `git add docs/PROJECT_CONTEXT.md`, коммит.
4. **Перед push** — проверка на утечку секретов в этом и любых других
   неотправленных коммитах (см. AAB-11, новое обязательное правило).
5. Перед закрытием задачи — спросить архитектора/пользователя напрямую
   «задачу закрываем?» (см. AAB-11, новый обязательный шаг), и только после
   подтверждения — заполнить итоговый контекст в задаче и запушить.

## Критерии готовности

- [ ] `docs/PROJECT_CONTEXT.md` в репозитории, содержимое — присланная
      архитектором версия, без самостоятельных правок содержания.
- [ ] Проверено отсутствие секретов в коммите перед push.
- [ ] Получено подтверждение закрытия задачи перед финальным push.
- [ ] Закоммичено и запушено на GitHub.

## Не делать в этой задаче

Не переписывать и не сокращать содержание документа — это зона архитектора,
не сеньора. Если что-то в документе кажется неточным относительно кода —
не исправлять самостоятельно, сообщить архитектору через пользователя.

## После выполнения (не для сеньора — для пользователя)

Если ещё не сделано — удалить старую вручную загруженную версию
`PROJECT_CONTEXT.md` из project knowledge на claude.ai (если она там
отдельным документом) — во избежание двух конфликтующих копий.

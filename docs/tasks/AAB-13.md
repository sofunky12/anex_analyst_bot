# AAB-13 — Зафиксировать урок: git push ≠ деплой на Fly

Чисто документационная задача, без изменений в коде.

## Контекст

При AAB-12 фикс с отзывом токенов какое-то время не проявлялся на проде,
хотя код был готов, закоммичен и запушен. Причина оказалась не в коде:
`fly deploy` собирает и разворачивает то, что лежит в локальной рабочей
директории **на момент вызова команды**, а не то, что в git — коммит и
push на это никак не влияют. Это отдельное, независимое действие. Сейчас
этот урок записан только в итоге задачи AAB-12 — рискует потеряться и
повториться.

## Что сделать

Добавить в `CLAUDE.md`, раздел «Платформенные ограничения», новый пункт:

- `fly deploy` разворачивает содержимое локальной рабочей директории на
  момент запуска команды — не состояние git (ни закоммиченное, ни
  запушенное). Значит: закоммиченный и запушенный код не обязательно уже
  работает на проде, пока явно не выполнен `fly deploy`; и наоборот — можно
  случайно задеплоить незакоммиченные изменения. Обнаружено при AAB-12:
  фикс не проявлялся на проде именно по этой причине. После любой задачи с
  изменениями в `src/` или `deploy/`, которые должны попасть на прод —
  явно выполнять `fly deploy` и не считать факт коммита/пуша признаком того,
  что изменения уже живые.
